# Аналитический пакет Miroshnikov для собственников

## Что находится в папке

- `miroshnikov_owner_dashboard.html` — готовый интерактивный отчёт на русском языке.
- `analysis_explorer.csv` — все поддерживаемые разрезы со знаменателями и уровнем надёжности.
- `insight_register.csv` — реестр выводов: доказательство → важность → интерпретация → действие.
- `period_summary.csv` — фиксированные периоды трафика.
- `behavior_modes.csv` — классификация визитов по сценариям поведения.
- `transitions.csv` и `routes.csv` — карта переходов и маршрутов.
- `product_scorecard.csv` — диагностика интереса к товарам.
- `retention.csv` и `cohort_retention.csv` — возвращаемость по зрелым окнам.
- `data_quality.csv` — реестр качества данных и трекинга.
- `run_manifest.json` — дата сборки, контрольные суммы и объёмы данных.
- `build_dashboard.py` — воспроизводимый сборщик отчёта.

## Как открыть отчёт

Дважды нажмите на файл `miroshnikov_owner_dashboard.html`. Сервер и подключение к интернету не нужны.

## Как пересобрать отчёт из новых выгрузок Logs API

```bash
python build_dashboard.py   --hits /путь/к/hits_full.csv   --visits /путь/к/visits_full.csv   --output ./build
```

Сборщик сохраняет только агрегаты. Сырые `clientID` и `visitID` в HTML не попадают.

## Как опубликовать

Для внутреннего доступа загрузите содержимое папки `build` на статический хостинг. Для постоянной эксплуатации лучше закрыть сайт корпоративной авторизацией.

## Ключевое ограничение

Прокси-намерение не является покупкой. Конверсия в покупку, выручка, CAC, ROAS и LTV останутся недоступны, пока не будут подключены заказы Shopify и стабильные события покупки.
